Otimização e padronização da base de dados, aumento de confiabilidade e eficiência de análises
Featured Project

Governança e Self-Service de Indicadores de Qualidade

Problema: Os dados de qualidade de software estavam dispersos e sem padronização, exigindo exploração manual recorrente e gerando baixa confiança nas métricas compartilhadas entre os times.
Decisão: Realizar a exploração via Jupyter Notebooks (AWS Glue) e Python para estruturar uma base técnica (spec) utilizando tabelas Apache Iceberg via Athena. O objetivo foi criar um dataset performático e governado para uso da comunidade de Dados.
Resultado: Implementação de uma fonte única de verdade que sustenta as visualizações no QuickSight, garantindo maior integridade e facilitando o acesso self-service aos indicadores de qualidade.
Sinal de sucesso: Disponibilização de datasets confiáveis para a comunidade interna e redução do tempo de resposta para visualização dos dados além de redução de custos e ganho de performance.
